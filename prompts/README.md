# Prompts

These prompts are starting points for local-agent workflows.

- `product-to-project.md`: convert a product brief into project slices.
- `engineering-agent.md`: execute a scoped engineering task.
- `qa-agent.md`: validate a completed change.

Prompts should include file scope, non-goals, and validation commands. Avoid
asking the local model to infer strategy from a vague request.
