# Objection Handling

| Objection | What It Usually Means | Response | Proof |
|-----------|------------------------|----------|-------|
| This seems risky | Buyer needs trust and validation | Start with a narrow pilot and explicit QA gate |  |
| We already have a process | Need to show integration, not replacement | Position as workflow acceleration |  |
| Local models are not good enough | Need task-fit framing | Explain scoped execution plus validation harness |  |
| Security is a concern | Needs architecture clarity | Explain local/private routing and review gates |  |
