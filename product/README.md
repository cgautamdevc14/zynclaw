# Product

Product owns the "why" and "what" before an agent touches the repo.

Use this folder to turn ideas into crisp, bounded work:

- `INTAKE.md`: first-pass problem capture.
- `PRD_TEMPLATE.md`: deeper requirements for larger work.
- `REQUIREMENTS_REVIEW.md`: readiness checks before engineering.
- `METRICS.md`: how success is measured.

## Product Rules

- Write the user problem before the solution.
- Name non-goals early.
- Prefer one narrow first slice over a broad platform rewrite.
- Every local-agent task needs acceptance criteria that QA can test.
