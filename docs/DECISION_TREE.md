# Decision Tree

## What Are You Trying To Do?

### I Have An Idea

Use:

- `product/INTAKE.md`
- Product intake issue form

Then ask:

- Who is this for?
- What problem does it solve?
- What is the smallest useful version?

### I Need To Coordinate Work

Use:

- `project/DELIVERY_PLAN.md`
- `project/RISK_REGISTER.md`

Then ask:

- Who owns it?
- Who reviews it?
- What can block it?

### I Want An Agent To Do Work

Use:

- `engineering/AGENT_CONTRACT.md`
- Local agent task issue form

Do not assign the task until it has:

- Scope.
- Non-goals.
- Acceptance criteria.
- Validation commands.
- Reviewer.

### I Need To Validate A Change

Use:

- `qa/ACCEPTANCE_CHECKLIST.md`
- QA report issue form

Then ask:

- What passed?
- What failed?
- What could regress?

### I Need To Launch Or Explain It

Use:

- `marketing/POSITIONING.md`
- `sales/SALES_PLAYBOOK.md`
- `templates/work-item/launch-handoff.md`

Then ask:

- What is the message?
- What proof supports it?
- What objections should be handled?

### I Need To Add Technology

Use:

- `technology/TECHNOLOGY_ROADMAP.md`
- `technology/STACK_DECISIONS.md`

Then ask:

- What problem does this solve?
- Who owns it?
- What command proves it works?
- What would make us remove it?

### I Need To Connect A Tool

Use:

- `connections/QUICK_START.md`
- `connections/CONNECTION_CATALOG.md`
- Connection request issue form

Then ask:

- What workflow needs this connection?
- Is read-only enough?
- Who owns the credential?
- What can the agent see or change?
- How do we reverse a bad action?
