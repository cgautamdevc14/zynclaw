# One-Page Overview

Zynclaw helps a team use local AI agents without turning the workflow into a
black box.

## What It Solves

Teams often ask agents to do work before the work is clear. That creates vague
outputs, missed validation, weak review, and confusion about who owns what.

Zynclaw fixes that by giving every function a simple place to write context
before work moves forward.

## The Workflow

```text
Idea -> Product brief -> Project plan -> Agent task -> QA -> Marketing/Sales -> Release
```

## Who Uses It

| Team | What They Own |
|------|---------------|
| Product | Problem, audience, requirements, acceptance criteria |
| Project | Owners, milestones, risks, status |
| Engineering | Implementation plan, agent contract, code review |
| QA | Validation, regression checks, release confidence |
| Marketing | Positioning, campaign, content, launch assets |
| Sales | ICP, talk track, demo flow, objections |

## What Good Looks Like

- Every task has a clear owner and reviewer.
- Every agent task has file scope and validation commands.
- Every launch has a QA check and customer-facing handoff.
- Every new technology has an owner and a reason to exist.
