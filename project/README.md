# Project

Project owns delivery shape, coordination, risks, and status.

Use this folder after product has defined the work and before engineering starts.

- `DELIVERY_PLAN.md`: milestones and slices.
- `RISK_REGISTER.md`: risks, owners, mitigations.
- `STATUS_REPORT_TEMPLATE.md`: weekly or per-release updates.
- `BACKLOG.md`: candidate work for local agents.

## Project Rules

- Every task has one owner and one reviewer.
- Every risk has a mitigation or an explicit acceptance decision.
- Work is split into reviewable slices.
- Agent-assigned tasks must include validation commands.
