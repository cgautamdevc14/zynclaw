# Contributing

Thanks for improving Zynclaw.

This repo is meant to stay useful for both technical and non-technical
teammates. Favor clear templates, practical checklists, and small changes over
large rewrites.

## Good Contributions

- Clearer product, project, QA, marketing, or sales templates.
- Better setup and validation docs.
- Safer agent workflow guardrails.
- Small scripts that reduce repeated manual work.
- Examples that show what good usage looks like.

## Before Opening A PR

- Run `make lint`.
- Keep generated or local files out of the diff.
- Update `docs/SOURCES.md` when changing source-backed technical defaults.
- Update `scripts/check_structure.py` when adding required folders or docs.

## Review Expectations

Every PR should explain:

- What changed.
- Who benefits.
- How it was validated.
- Any risks or follow-up work.

Agent-generated changes still need human review.
