# Release Checklist

## Before Merge

- [ ] PR description explains user impact.
- [ ] Validation commands pass.
- [ ] QA checklist complete.
- [ ] Rollback path is known.
- [ ] Docs updated if behavior changed.

## After Merge

- [ ] Smoke checks pass.
- [ ] Monitoring reviewed.
- [ ] Team notified when relevant.
- [ ] Follow-up tasks created.
