# Acceptance Checklist

## Stack

- [ ] `make doctor` has no failures.
- [ ] `make acceptance` passes on the target endpoint.
- [ ] LiteLLM route points at the intended model.

## Task

- [ ] Objective is satisfied.
- [ ] Acceptance criteria are met.
- [ ] Non-goals were respected.
- [ ] Validation commands were run.
- [ ] Known failures are documented.

## Review

- [ ] Human code review complete.
- [ ] Product signoff complete when behavior changed.
- [ ] QA signoff complete for release-bound work.
